Plot Model Parameter

PMP is a file extension for a plotter model parameter file used by PC3 files.
Since PC3 depend on the HDI that is closed source, undocumented and compatible only under Windows S.O. I'm not very interested to understand how PC3 and PMP works.
Maybe can be useful understand how PMP works for port paper size to system printer directly, without an abstraction software level.
So, the most interesting part of the PMP file is udm{ taht stands for UserDefinedMetric.

udm{

Stands for User Defined Metric, the custom paper size.
This is the start of interesting part.

calibration{

Calibration.

Default value is _x=1.0\n _y=1.0\n }


media{

Start of media.


size{

Start of size name and other info.

An example can be:
0{
caps_type=2
name="UserDefinedMetric (1470.00 x 297.00mm)
localized_name="7x1 MODULI (1470.00 x 297.00 mm)
media_description_name="UserDefinedMetric Orizzontale 1470.00L x 297.00A - (0, 0) x (1470, 297) =436590 mm
media_group=15
landscape_mode=TRUE
}
Of this imho the most interesting part is localized_name and maybe landscape_mode.
All other info can be readed from other lines.

description{

Start of descriotion size.

An example can be:
0{
caps_type=2
name="UserDefinedMetric Orizzontale 1470.00L x 297.00A - (0, 0) x (1470, 297) =436590 mm
media_bounds_urx=1470.0
media_bounds_ury=297.0
printable_bounds_llx=0.0
printable_bounds_lly=0.0
printable_bounds_urx=1470.0
printable_bounds_ury=297.0
printable_area=436590.00000 ( +BE&D%#\@)
dimensional=TRUE
}
This are the dimension of the paper sizes (media_bounds_urx/y) and the printable sizes (printable_bounds_urx/y).
The printable_area value is simply printable_bounds_urx * printable_bounds_ury.
All the sizes are always expressed in mm not in "points", most printer use "point" (for example all PPD files).
1 "point" is about 0.3527 mm.
